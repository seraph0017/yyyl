4. 风险与缓解

风险 影响 缓解
资金并发导致余额错误 高 行级锁、唯一约束、账户非负约束、幂等流水
退款状态影响旧订单列表 高 状态分层，旧接口兼容层只做桥接
导出泄露敏感数据 高 私有目录、鉴权下载、手机号默认脱敏
小程序码微信接口限流 中 access_token 缓存、按错误码重试、批量生成异步化
分类 tabBar 参数传递失效 中 storage + switchTab + onShow 消费

5. 上线前检查

v1.7 已完成实现与本地验证。进入发布前，按以下顺序做生产前检查：

  • 审阅并执行 Alembic 迁移，确认新增表、索引、唯一约束和账户非负约束符合生产数据库状态。
  • 配置微信小程序码生成所需的各营地 AppID/AppSecret，确认二维码图片目录可写且不暴露私有导出目录。
  • 发布后优先验证二维码解析、订单导出下载、支付入账、订单完成结算、退款回调五条闭环。
