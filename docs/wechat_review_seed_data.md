# 微信审核测试数据准备说明

## 目标

为微信小程序审核准备最小可用线上测试数据，让审核人员可以进入小程序浏览首页、商品列表、商品详情并创建待支付订单。

## 数据范围

`server/seed_xijiao_demo_data.py` 会补齐 `site_id=1` 西郊林场的审核展示数据：

- 商品：营位、活动、装备租赁、营地小店、周边商品
- SKU：饮料、泡面、T 恤等多规格商品
- 定价规则：工作日、周末、节假日价格
- 库存：未来 45 天营位库存
- 首页配置：首页 Banner 和公告
- 优惠规则：连住折扣、多人折扣

脚本不会创建订单、票券、支付流水或退款数据，避免污染线上经营报表。

## 执行方式

线上服务器进入后端目录后执行：

```bash
cd /path/to/yyyl/server
conda activate yyyl
python seed_admin.py
python seed_xijiao_demo_data.py
```

如果线上使用容器运行 API，需要在能访问同一套数据库配置的环境中执行以上脚本。

## 后台测试账号

执行 `seed_admin.py` 后会创建或重置后台管理员：

```text
用户名：admin
密码：admin123456
```

审核提交时如需要后台测试账号，可提供该账号。正式上线后应立即修改密码或禁用该账号。

## 幂等性和安全边界

- `seed_xijiao_demo_data.py` 按商品名称、SKU 编码、页面 key、规则条件做 upsert，可重复执行。
- 脚本不会清空任何表。
- 已存在的库存会保留已售和锁定数量，只按 `total - sold - locked` 重算可售数量。
- 当前脚本只补 `site_id=1`。如果审核版本使用大聋谷或其他营地，需要另行扩展对应 `site_id` 数据。
