## 一、日常露营票订票（3.1.1）

### 1.1 基础预定流程

| 字段 | 内容 |
|------|------|
| **用例编号** | TC-CAMP-001 |
| **所属模块** | 日常露营票 — 基础预定全流程 |
| **优先级** | P0 |
| **前置条件** | 用户已登录；营位商品已上架且有可用库存；管理员配置身份登记为"必填" |
| **测试步骤** | 1. 进入"日常露营"分类页<br>2. 选择一个"过夜露营票"商品<br>3. 在详情页选择入住日期（如周六-周日，两天一夜）<br>4. 点击"立即预定"<br>5. 填写身份信息（姓名、身份证号、手机号）<br>6. 签署免责声明<br>7. 确认订单信息（含价格、优惠、身份信息）<br>8. 点击"支付"进入模拟支付页面<br>9. 点击"模拟支付成功" |
| **预期结果** | 1. 订单状态变为"已支付/待使用"<br>2. 生成电子票（含二维码）<br>3. 对应日期库存减1<br>4. 收入进入"待确认"账户<br>5. 用户收到"订单支付成功"通知<br>6. OrderItem按天拆分存储（两天一夜生成2条记录） |
| **测试类型** | 功能 |

## 附录B：支付全流程测试（沙箱）

| 序号 | 测试场景 | 测试步骤 | 预期结果 |
|------|---------|---------|---------|
| 1 | 模拟支付成功 | 下单→点击"模拟支付成功" | 订单→已支付；库存已售；回调触发 |
| 2 | 模拟支付失败 | 下单→点击"模拟支付失败" | 订单保持待支付；库存保持锁定 |
| 3 | 超时后支付 | 等待超时→再次支付 | 订单已取消，无法支付 |
| 4 | 重复支付 | 已支付订单再次调用支付接口 | 返回"订单已支付"，不重复扣款 |
| 5 | 退款模拟 | 已支付订单→申请退款→审批通过 | 订单→已退款；资金返还 |
| 6 | 次数卡0元支付 | 次数卡预定 | 不走支付流程，直接已支付 |
| 7 | 年卡免费预定 | 年卡权益预定 | 不走支付流程，直接已支付 |
| 8 | 积分兑换0元 | 积分兑换 | 不走支付流程，扣积分 |
| 9 | 支付策略切换 | 管理员切换模拟↔真实支付 | 需管理员密码二次确认；切换后新订单使用新策略 |
