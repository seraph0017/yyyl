1. 概述

本次后端开发范围覆盖 v1.7 的 P0 主链路：

  1. 小程序码系统：商品、分类、活动、CMS 自定义页面生成小程序码，扫码解析与归因日志。
  2. CMS 页面二维码：复用 v1.6 CmsPage，为已发布页面生成二维码。
  3. 订单高级筛选与导出：扩展订单查询参数，提供私有导出任务和鉴权下载。
  4. 资金结算：支付成功进入 pending_amount，订单完成后结算到 available_amount。
  5. 退款策略升级：新增 RefundRecord 主事实表，支持全额/部分/按项退款，支持保留订单或取消订单。

不在本端范围内：

  • 管理后台页面与交互实现。
  • 小程序扫码分发页与本地归因存储。
  • 商家小程序版后台 P2 首期实现。

POST /admin/qrcodes 请求：

  "target_key": "123",

安全硬约束：

  • path 只能由后端枚举模板生成，不允许从数据库任意字符串直接返回。
  • 微信返回必须校验 Content-Type=image/* 和文件大小；错误 JSON 不得保存为图片。
  • access_token 必须缓存，限流/网络错误可重试，参数错误直接失败。

4.3 资金结算

方法 路径 说明
GET /api/v1/admin/finance/settlements 结算记录列表
POST /api/v1/admin/orders/{id}/settle 手动触发结算，super_admin
GET /api/v1/admin/finance/overview 增加结算统计

支付成功处理：

  1. 锁定 Order。
  2. 以微信 transaction_id/payment_no 幂等检查。
  3. FinanceAccount.pending_amount += order.actual_amount。
  4. 创建 FinanceTransaction(type='income', account_type='pending')。

结算处理：

  1. 仅处理 Order.status='completed' 且 settlement_status != 'settled'。
  2. 同一事务内 SELECT ... FOR UPDATE 锁定 Order 与 FinanceAccount。
  3. settle_amount = actual_amount - refunded_amount - settled_amount。
  4. pending_amount -= settle_amount，available_amount += settle_amount。
  5. 创建 FinanceTransaction(type='settlement', from_account='pending', to_account='available')。

创建请求：

{
  "refund_mode": "partial",
  "order_action": "keep_order",
  "refund_amount": "50.00",
  "release_inventory": false,
  "reason": "服务体验补偿",
  "items": [
    { "order_item_id": 1001, "refund_amount": "50.00", "quantity": 1 }
  ],
  "confirm_code": "123456"
}

状态分层规则：

场景 Order.status payment_status refund_status
部分退款不取消 不改变服务状态 partial_refunded partial
全额退款不取消 不改变服务状态 refunded refunded
全额退款并取消 cancelled refunded refunded
部分退款并取消 cancelled partial_refunded partial

全额退款不取消订单只能 super_admin 操作，必须二次确认并强制备注。

5. 服务层关键约束

5.1 二维码归因

  • 小程序扫码解析成功后会在下单时传 source_qrcode_id/source_channel/source_scanned_at。
  • 后端必须校验二维码存在、active、site_id 匹配。
  • 归因 TTL 默认 24 小时，超过 TTL 的来源不写入订单。

5.2 导出任务

  • 5000 行以内可同步生成任务并返回 completed。
  • 超过 5000 行写入 pending，由 Celery 生成。
  • 导出字段默认手机号脱敏；完整手机号需要 order export 权限。
  • OperationLog 记录 filters、row_count、file_format、created_by。

5.3 退款与财务

  • 未结算订单退款优先扣 pending_amount。
  • 已结算订单退款扣 available_amount；不足时审批失败。
  • 部分结算订单优先扣 pending，不足扣 available。
  • 微信退款成功回调以 out_refund_no/refund_no 幂等更新 RefundRecord。
  • 回调未到的 processing 记录由定时任务查询补偿。
