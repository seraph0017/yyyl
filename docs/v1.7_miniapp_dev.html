1. 概述

小程序端本期负责二维码扫码后的用户路径和订单来源归因：

  1. 新增统一扫码分发页 /pages/qr/landing。
  2. 调用后端解析 scene，按目标跳转商品、分类、活动商品、CMS 活动页、自定义页面。
  3. 将 qrcode_id/channel/scanned_at 写入本地归因缓存，默认 TTL 24 小时。
  4. 下单时把有效归因信息带给后端，写入订单来源。
  5. 强化 CMS 页面错误态，支持自定义页面二维码扫码访问。

不在本端范围内：

  • 小程序码图片生成。
  • 管理后台二维码管理、导出、退款审批。
  • 商家小程序版后台 P2。

6. 来源归因工具

utils/attribution.ts：

import type { OrderAttribution, QrcodeResolveResponse } from '@/types/qrcode'

const STORAGE_KEY = 'yyyl:order_attribution'

export function saveOrderAttribution(data: QrcodeResolveResponse) {
  const now = Date.now()
  const ttl = (data.attribution_ttl_seconds || 86400) * 1000
  const payload: OrderAttribution = {
    qrcode_id: data.qr_code_id,
    channel: data.channel || 'default',
    scanned_at: new Date(now).toISOString(),
    expires_at: new Date(now + ttl).toISOString(),
  }
  uni.setStorageSync(STORAGE_KEY, payload)
}

export function getValidOrderAttribution(): OrderAttribution | null {
  const payload = uni.getStorageSync(STORAGE_KEY) as OrderAttribution | ''
  if (!payload || typeof payload === 'string') return null
  if (Date.parse(payload.expires_at) <= Date.now()) {
    uni.removeStorageSync(STORAGE_KEY)
    return null
  }
  return payload
}

export function clearOrderAttribution() {
  uni.removeStorageSync(STORAGE_KEY)
}

规则：

  • TTL 默认 24 小时。
  • 重复扫码以后一次为准。
  • 未登录用户也保存本地归因，下单时再带上。
  • 下单成功后不立即清除，避免同一活动页连续下多单丢失来源；TTL 到期自动失效。

7. 扫码分发页

pages/qr/landing.vue 核心逻辑：

import { onLoad } from '@dcloudio/uni-app'
import { resolveQrcode } from '@/api/qrcode'
import { saveOrderAttribution } from '@/utils/attribution'

onLoad(async (options) => {
  const scene = decodeURIComponent(options?.scene || options?.q || '')
  if (!scene) {
    showError('二维码参数无效')
    return
  }

  try {
    const data = await resolveQrcode(scene)
    if (data.status !== 'active') {
      showError('该二维码已停用')
      return
    }
    saveOrderAttribution(data)
    navigateByTarget(data)
  } catch {
    showError('二维码解析失败，请稍后重试')
  }
})

function navigateByTarget(data) {
  const path = data.path
  if (!isAllowedPath(path)) {
    showError('二维码目标无效')
    return
  }
  uni.redirectTo({ url: path })
}

function isAllowedPath(path: string) {
  return [
    '/pages/product-detail/index',
    '/pages/category/index',
    '/pages/cms-page/index',
  ].some(prefix => path.startsWith(prefix))
}

UI 状态：

  • 加载中：显示“一月一露 正在打开...”。
  • 失败：显示错误原因和“返回首页”按钮。
  • 停用：显示“该二维码已停用”。

8. 跳转目标

target_type 跳转
category /pages/category/index?category={target_key}，使用 switchTab 后通过事件/Storage 传参，或改为非 tab 页面入口
product /pages/product-detail/index?id={target_key}
activity_product /pages/product-detail/index?id={target_key}
activity_page /pages/cms-page/index?page_code={target_key}
custom_page /pages/cms-page/index?page_code={target_key}

注意：pages/category/index 是 tabBar 页面，uni.redirectTo 不能跳 tabBar。实现时优先：

  1. 将 category 参数写入 storage yyyl:pending_category_filter。
  2. uni.switchTab({ url: '/pages/category/index' })。
  3. 分类页 onShow 读取并应用筛选后清除该 storage。

9. 创建订单携带来源

import { getValidOrderAttribution } from '@/utils/attribution'

const attribution = getValidOrderAttribution()
const payload = {
  ...orderPayload,

  source_channel: attribution?.channel,
  source_scanned_at: attribution?.scanned_at,
}

后端会校验二维码有效性；小程序只负责传递，不在本地信任来源。

10. CMS 页面错误态

修改 pages/cms-page/index.vue：

  • 404/CMS_PAGE_NOT_FOUND：页面不存在或未发布。
  • CMS_PAGE_INACTIVE：页面已停用。
  • 网络错误：显示重试。
  • 分享 path 保持 /pages/cms-page/index?page_code={pageCode}。

扫码进入 CMS 页面时，不使用 preview_token；预览仍只来自 Admin 生成的预览链接。
