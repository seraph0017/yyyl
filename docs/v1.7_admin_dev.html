1. 概述

管理后台本期负责四类 P0 能力：

  1. 小程序码管理：商品、分类、活动、CMS 页面生成、查看、下载、停用二维码。
  2. 订单高级筛选与导出：多条件筛选、筛选摘要、导出任务抽屉、私有下载。
  3. 资金结算展示：财务概览展示待确认/可提现/结算记录，订单详情展示结算状态。
  4. 退款策略升级交互：统一退款弹窗、全额/部分/按项退款、保留订单/取消订单、高危确认。

不在本端范围内：

  • 微信小程序码生成服务、导出文件生成、资金结算业务逻辑。
  • 小程序扫码分发页与来源归因本地存储。
  • 商家小程序版后台 P2 页面。

4. API 模块

4.1 api/qrcode.ts

import { get, post, patch } from '@/utils/request'
import type { MiniProgramQrcode, QrcodeCreateRequest } from '@/types/qrcode'
import type { PaginatedResponse } from '@/types'

export function createQrcode(data: QrcodeCreateRequest) {
  return post<{ data: MiniProgramQrcode }>('/admin/qrcodes', data)
}

export function getQrcodes(params: {
  page?: number; page_size?: number; target_type?: string; channel?: string; status?: string
}) {
  return get<{ data: PaginatedResponse<MiniProgramQrcode> }>('/admin/qrcodes', params)
}

export function regenerateQrcode(id: number) {
  return post<{ data: MiniProgramQrcode }>(`/admin/qrcodes/${id}/regenerate`)
}

export function updateQrcodeStatus(id: number, status: 'active' | 'inactive') {
  return patch(`/admin/qrcodes/${id}/status`, { status })
}

export function getQrcodeDownloadUrl(id: number) {
  return `/api/v1/admin/qrcodes/${id}/download`
}

4.2 api/order-export.ts

export function createOrderExport(data: {
  filters: OrderSearchParams
  file_format: 'xlsx' | 'csv'
  include_sensitive?: boolean
}) {
  return post<{ data: OrderExportTask }>('/admin/orders/export', data)
}

export function getOrderExportTasks(params?: { page?: number; page_size?: number; status?: string }) {
  return get<{ data: PaginatedResponse<OrderExportTask> }>('/admin/orders/export-tasks', params)
}

export function downloadOrderExportTask(id: number) {
  return get(`/admin/orders/export-tasks/${id}/download`, undefined, { responseType: 'blob' })
}

4.3 api/refund.ts

export function createRefund(orderId: number, data: RefundCreateRequest) {
  return post<{ data: RefundRecord }>(`/admin/orders/${orderId}/refunds`, data)
}

export function getOrderRefunds(orderId: number) {
  return get<{ data: RefundRecord[] }>(`/admin/orders/${orderId}/refunds`)
}

export function approveRefundRecord(id: number, data: { confirm_code?: string; operation_password?: string }) {
  return post(`/admin/refunds/${id}/approve`, data)
}

export function rejectRefundRecord(id: number, data: { reason: string }) {
  return post(`/admin/refunds/${id}/reject`, data)
}

5. 页面设计

5.1 小程序码管理页

路径：views/qrcode/index.vue

功能：

  • 筛选：目标类型、状态、渠道、关键词。
  • 表格列：二维码缩略图、目标类型、标题、渠道、状态、扫码次数、最近扫码、生成时间。
  • 操作：预览、下载、重新生成、停用/启用。
  • 新建弹窗：目标类型切换后展示对应选择器。

目标选择器规则：

target_type 选择器
category 品类下拉，来源 Product.category 枚举
product 商品远程搜索
activity_product 活动商品远程搜索
activity_page/custom_page CMS 页面搜索，仅允许 active 且已发布

5.2 CMS 页面列表二维码入口

修改 views/cms/index.vue：

  • 操作列新增“二维码”按钮。
  • current_version_id 为空时禁用，tooltip 提示“发布后可生成二维码”。
  • 点击后调用 POST /admin/cms/pages/{id}/qrcode，弹出二维码预览。

5.3 订单高级筛选

组件：components/orders/OrderAdvancedFilter.vue

布局：

  • 第一行常用条件：关键词、订单状态、支付状态、下单时间、搜索/重置。
  • 折叠高级区：退款状态、商品分类、商品、SKU、预定日期、支付时间、金额区间、核销状态、二维码渠道。
  • 底部显示筛选摘要，例如：“已筛选：已支付、2026-06-01 至 2026-06-18、渠道 poster”。
  • 保存筛选条件到 localStorage，key=yyyl:orders:filters:{site_id}。

5.4 导出任务抽屉

组件：components/orders/OrderExportDrawer.vue

流程：

  1. 用户点击“导出”。
  2. 弹窗显示当前筛选摘要、预计导出字段、文件格式，默认 .xlsx。
  3. 勾选“完整手机号”时检查权限，无权限禁用并提示。
  4. 创建任务后打开抽屉，轮询任务状态。
  5. completed 后通过鉴权接口下载 blob。

导出下载不得使用公开 URL，不要把下载链接直接拼成静态地址。

5.5 统一退款弹窗

组件：components/orders/RefundDialog.vue

字段：

字段 控件 规则
refund_mode segmented/radio full/partial/item
order_action radio keep_order/cancel_order
refund_amount input-number full 自动填可退金额，partial 可改
items table + input-number item 模式展示订单项
release_inventory switch cancel_order 默认 true，keep_order 默认 false
reason textarea 必填
confirm_code/password 高危时展示 ≥500、偏差>20%、全额退款不取消

高危规则：

  • 全额退款不取消订单：仅 super_admin 可操作，强制 operation password。
  • 自定义金额偏差超过 20%：提示红色风险说明。
  • 已核销订单取消：默认不释放库存，super_admin 可强制。

5.6 财务结算展示

修改 views/finance/index.vue：

  • 概览卡增加“不可提现金额”“可提现金额”“本月结算金额”“结算失败数”。
  • 交易流水增加 settlement 类型筛选。

新增 views/finance/settlements.vue：

  • 表格：订单号、结算金额、状态、触发方式、结算时间、失败原因。
  • 操作：失败记录可手动重试（super_admin）。
