2. 结论

后端开发文档达到可开发标准。架构评审中的 5 个 HIGH 问题均已在后端方案中落到具体约束：

  • 资金结算：行级锁、唯一约束、非负约束、Celery 补偿。
  • 退款状态机：服务状态、支付状态、退款状态分层。
  • 二维码归因：下单来源字段、TTL、后端校验。
  • 导出安全：私有目录、鉴权下载、敏感字段权限。
  • 微信小程序码：access_token 缓存、限流重试、path 白名单。

3. 必须在实现阶段注意

编号 要求
BE-01 Alembic 迁移中必须同时创建索引、唯一约束和账户非负约束
BE-02 Order.status 不要再承载退款语义，退款语义必须走 refund_status/payment_status
BE-03 导出文件不得落到 /images 或任何公开静态目录
BE-04 微信退款回调和主动查询补偿必须共享同一幂等更新函数
