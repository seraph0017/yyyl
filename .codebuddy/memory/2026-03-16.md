## 订单确认页 ¥NaN 修复

### 问题
- `order-confirm/index.vue` 的 `loadData()` 直接将 API 原始返回 `get<IProduct>(...)` 当作 `IProduct` 使用
- API 返回的字段是 `base_price`（字符串），没有 `current_price` 字段
- 导致 `p.current_price` 为 `undefined`，计算金额得到 `NaN`
- 控制台报错 `TypeError: Cannot read property 'toFixed' of undefined`

### 修复
- 将 API 返回类型改为 `Record<string, any>`，手动映射为 `IProduct` 结构
- `parseFloat(raw.base_price) || 0` 确保价格为有效数字
- 同时映射 category（处理 rental→equipment_rental、shop→camp_shop）

## 日历最大连续预定天数限制

### 问题
- 日历范围选择没有天数上限，用户可以连选8晚甚至更多
- PRD 3.2.1 年卡会员规则：MAX_CONSECUTIVE = 5（最大连续天数），但前端没有校验

### 修复
- **product-detail/index.vue** 新增 `MAX_BOOKING_NIGHTS = 5` 常量
- 在 `onSelectDate` 中，设置离店日前先校验 `daysBetween(checkIn, checkOut) > MAX_BOOKING_NIGHTS`
- 超出时 `uni.showToast({ title: '最多连续预定5晚' })` 阻止选择
- 新增 `daysBetween()` 辅助函数计算天数差

## 商品介绍 HTML 标签暴露修复

### 问题
- 商品详情页"商品介绍"区域直接显示了原始 HTML 标签 `<p>...</p>`
- 原因：使用 `<text>{{ product.description }}</text>` 插值，HTML 被当作纯文本渲染

### 修复
- **product-detail/index.vue** 第132行：`<text>` → `<rich-text :nodes="product.description">`
- `rich-text` 组件会正确解析和渲染 HTML 内容
