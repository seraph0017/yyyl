## 修复数据库表缺失导致的 500 错误
campsites 接口返回 500 Internal Server Error，根本原因：
1. **`product` 表缺少 `seckill_warmup_hours` 列** — v1.5 模型新增但未同步到数据库 → 通过 ALTER TABLE 添加
2. **11 张 v1.5 新增表未创建** — bundle_config, bundle_item, camp_map, camp_map_zone, expense_request, expense_type, mini_game, performance_config, performance_detail, performance_record, product_ext_insurance → 通过 `Base.metadata.create_all` 批量创建
3. `Product` 模型的 `ext_insurance` 关系使用 `lazy="selectin"`，查询 Product 时自动加载不存在的 `product_ext_insurance` 表导致 500

## 修复数据库列缺失导致的 orders 500 错误
全面扫描所有表，发现并修复 4 个表的列缺失：

- **`order_item` 表**：+`bundle_group_id`(VARCHAR), +`bundle_config_id`(BIGINT), +`is_bundle_item`(BOOLEAN)

全部通过 ALTER TABLE ADD COLUMN IF NOT EXISTS 修复，至此所有模型定义的列已与数据库同步

## 修复 GET /admin/products 405 Method Not Allowed
- **原因**：后端 `products.py` 没有 `GET /api/v1/admin/products` 管理端商品列表接口，只有 POST（创建）。前端 admin 的 bundle/camp-map/seckill 页面都调用 `GET /admin/products` 获取商品列表
- **修复**：在 `server/routers/products.py` 新增 `GET /api/v1/admin/products` 接口（复用 `product_service.list_products`，不默认过滤状态，按 site_id 过滤）
- **前端**：`admin/src/api/product.ts` 的 `getProducts` 从 `'/products'`(C端) 改为 `'/admin/products'`(B端)

## 修复 GET /admin/products 422 Unprocessable Entity
- **原因**：`PaginationParams.page_size` 限制 `le=100`，前端 bundle/camp-map/seckill 页面请求 `page_size=200` 超过限制导致验证失败
- **修复**：在 `schemas/common.py` 新增 `AdminPaginationParams`（`page_size` 上限 500），管理端商品列表接口改用此参数类
