技术选型：确定适配微信小程序的前后端技术栈（比如前端原生 / Uni-app、后端 SpringBoot/Node.js、数据库 MySQL+Redis），评估服务器、域名、备案要求；
架构与数据库设计：输出前后端分离架构图、核心数据库表结构（用户、营地、库存、订单、支付、营销等）、RESTful 接口规范文档；
核心难点方案：针对露营地小程序的核心痛点，输出可落地的技术方案：预约库存锁 / 解锁逻辑、微信支付全流程对接方案、平台型项目的分账逻辑、小程序原生接口对接方案、数据安全方案；
合规技术设计：输出用户隐私合规设计、数据存储合规方案、支付安全合规方案，匹配微信小程序的审核规则；
技术评审：评审前后端开发 Agent 的代码，确保符合架构规范，解决突发技术难点。
核心输出：技术选型报告、系统架构图、数据库设计文档、接口规范文档、核心业务技术方案、支付对接技术方案、合规技术方案
IDE 工作边界：独立会话，不直接编写业务代码，仅负责技术方案设计与评审。
