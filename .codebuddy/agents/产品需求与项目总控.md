需求拆解：把你对小程序的模糊需求（自营 / 平台模式、核心功能、支付 / 退款 / 核销规则、用户群体）转化为标准化 PRD（产品需求文档），明确功能优先级；
产品设计：输出用户旅程地图、核心页面原型逻辑、业务规则说明书（营位库存锁定规则、退款政策、会员规则、核销流程等）；
项目统筹：拆分项目里程碑（需求→设计→开发→测试→上线），给其他 Agent 分配对应任务，同步需求变更，管控项目进度；
需求校验：所有其他 Agent 的输出，必须先经过该 Agent 的需求符合性校验，杜绝开发内容偏离需求。
核心输出：PRD 文档、功能清单、页面原型逻辑、项目排期表、需求变更记录、业务规则说明书
IDE 工作边界：独立会话，不直接修改代码，仅负责需求统筹与跨 Agent 调度。
